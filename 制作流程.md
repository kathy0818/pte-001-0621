# PTE 模考整理卷 · 制作流程(新考试·交接手册)

> **这份是给"下一次会话"的操作手册。** 场景:我又考了一次 PTE 模考,
> 有了**新的 GitHub 仓库**、**新的截图 PDF**(题目/作答 + 详解)、**新的成绩 PNG**。
> 你(新会话)拿到这些,照着这篇**从头到尾**就能把"错题与详解·整理卷"做出来。
>
> 配套文档:`README.md`(项目总览 + 版本规则)。本篇是**详细动手流程**。

---

## ⭐ 先记住三条铁律(最重要)

1. **截图 PDF 没有文字层 —— 必须渲染成图片用眼睛读。**
   `pdftotext` / `build/extract/*.txt` 基本是空的,**别依赖文字提取**。一切靠把 PDF 渲成 PNG、放大、肉眼逐字誊录。
2. **一切以原图为准,人工对照誊录。** 分数、批改的每个词、AI 中文建议,都要和截图核对到一致;拿不准就放大再看。
3. **成品要版本化、旧版永不改。** 成品 = 一对 `(pdf, tex)`,放进唯一的 `整理卷/` 文件夹,**每个 part 各自从 `000` 起** +1;纯文字版本单文件自包含,含图片版本做成子文件夹(tex+img+pdf)。详见末尾 §A。

## 2. 把截图"读"出来(核心技巧)

PDF 是图,没有文字。读法从粗到细:

**① 先看整体结构**
```bash
pdfinfo 口语详解1.pdf | grep -iE "pages|page size"   # 常是 A3,一页里叠了几张手机截图
pdftoppm -png -r 130 口语详解1.pdf /tmp/d            # 整本渲成 /tmp/d-1.png …
```
然后**逐页看图片**(用看图工具 / Read 图片),建立"第几页有第几题、版面怎么排"的地图。

**② 小字读不清就放大**(分数、AI 建议、彩色批改词最容易糊):
```bash
# build/zoom.py <pdf> <0基页号> x0 y0 x1 y1 [dpi] [名字]
# 坐标是页面比例 0~1,框出你要看的那一小块;输出到 build/zoom/<名字>.png
python3 build/zoom.py 口语详解1.pdf 0  0.30 0.10 0.70 0.30  400  q1_score
```

**③ 想自动把灰底卡片裁出来 / 长截图切片**:
```bash
python3 build/render.py    口语1.pdf 0        # 自动裁卡片+切片 → build/pages/
python3 build/crop_tool.py 口语1.pdf 0 350 /tmp/c.png   # 紧贴内容裁一刀
```

**④ 逐字誊录**,每题都要读准并分清:
- **题干 / 原文**(prompt / source text)
- **范文**(若有,WE 邮件、部分口语题平台给参考)
- **你的作答**(详解里有你的录音转写 / 你写的原文)
- **逐项评分**(每个维度的得分,如 Content/Grammar/Spelling…)
- **彩色批改**:红=改/拼写、蓝=删、紫=插、灰=漏读;具体是哪个词
- **AI 中文建议**(原文照抄)

## 3. 各部分要保留什么(按题型)

> 题型以**详解 PDF 实际内容**为准,下面是 PTE Core 常见结构与"每题至少保留的要素"。

- **写作 Writing**
  - **SWT 概括写作**(每题满分 8):原文 → 你的作答+批改 → 评分(Content/Form/Grammar/Vocabulary)→ AI 建议。
  - **WE 邮件写作**(每题满分 15):题干(含三个 themes)→ **范文** → 你的作答+批改 → 评分(Content/Form/Email Conventions/Organization/Vocabulary/Grammar/Spelling)→ AI 建议。
- **口语 Speaking**(RA 朗读 / RS 复述句子 / DI 描述图 / RTS 情景应答 / ASQ 简答 等)
  - 题干或原文 → **你的录音转写(AI 语音识别)** → 评分(RA/RS/DI: 内容·发音·流利度;RTS: 恰当性·发音·流利度,均 /90,内容多 /3 或 /5)。
  - ⭐ **平台用了颜色就一定要复刻**:AI 语音识别是**逐词上色**的——{绿=优 / 橙=良 / 红=差 / `/`=停顿 / 红删除线=漏读}。用 `ptestyle.sty` 里的 `\sg \sy \sr \pse \smiss` + `\asrlegend` + `asrbox` 还原,**不要只写文字摘要**(这点上一版偷懒被指出过)。
  - ⭐ **DI 配图 + DI/RTS 范文在「题目卷」里,不在「详解卷」**:详解卷(`口语详解1.pdf`)里图被弹窗盖住;去 `口语1/2/3.pdf`(列表视图)能找到**清晰大图**和平台**范文/参考答案(答案:)**。用 `build/zoom.py` 裁图存到 `build/paper/img/`,正文 `\includegraphics`。
  - 含图片 → 成品做成子文件夹(见 §6/§A)。
- **阅读 Reading** / **听力 Listening**
  - 题型(选择/填空/排序/听写等)→ **正确答案 vs 你的作答** → 得分 → 解析。
  - 填空/听写类逐空标对错;选择类标你选的 vs 正确项。

## 4. 在 `build/paper/` 里排版(复用现成样式)

**不要重写样式**——复用 `build/paper/ptestyle.sty`(配色、批改宏、题块、评分表、章节标题、图例都在里面)。

**封面 `cover.tex`**:改这几处,数字来自**成绩 PNG**:
- 提交时间、**总分 Overall**、四项分数;
- 四项条形图**条长 = 得分 / 90**(上一套:听 33 / 读 42 / 说 29 / 写 57,对应条长比例)。

**每个部分两个文件**:
- 正文 `sec_<英文>_body.tex`(如 `sec_speaking_body.tex`)——逐题内容;
- 主文件 `preview_<英文>.tex`:
  ```latex
  \documentclass[11pt]{article}
  \usepackage{ptestyle}
  \begin{document}
  \input{cover.tex}
  \input{sec_<英文>_body.tex}
  \end{document}
  ```

**批改宏速查**(全在 `ptestyle.sty`):

| 宏 | 含义 | 外观 |
|----|------|------|
| `\fix{原}{正}` | 修改 / 拼写 / 缺空格 | 红删除线 + 绿改正 |
| `\del{词}` | 应删除 | 蓝删除线 + "删" |
| `\ins{词}` | 应插入 | 紫色 + "∧插" |
| `\add{原}{正}` | **平台漏标、本卷补标** | 橙波浪线 + "补" |
| `\miss{词}` | 口语漏读 / 未作答 | 灰删除线 |
| `\spk{颜色}{词}` | 发音好/中/差 | 彩色字 |
| `\sfull/\spart/\szero{x / y}` | 评分:满分/部分/零分 | 绿/橙/红 |

结构件:题块 `\begin{qblock}{标号·题型}{右上:得分/用时} … \end{qblock}`;
文本框 `promptbox`(题干/原文,灰)`modelbox`(范文,绿)`youbox`(你的作答,红)`notebox`(注解,蓝);
小标题 `\ptesub{…}`;评分表 = `tabularx` + `\scorehead`(用法照抄 `sec_writing_body.tex`)。

> 💡 **增值点**:仔细核对原图,若平台**漏标**了明显错误(上套写作里 `replay→reply`、缺动词就是漏标),用 `\add` 橙色"补"标出来。

> 原图若在分页处截断、缺了内容,照实标注(参考写作 Q4 用红字注明"原图未截全"),**不要瞎补**。

## 6. 出版本(套版本规则)

成品要变成**单文件自包含 tex** + PDF,放进**唯一的成品文件夹 `整理卷/`**,序号全局递增。

**① 展平**——用现成脚本 `build/flatten.py`(读取 `ptestyle.sty + cover.tex + sec_<part>_body.tex` 内联成成品):

```bash
ls 整理卷/                                  # 看这个 part 现有最大序号,本次用下一个

# 纯文字: python3 build/flatten.py <part英文> <中文> <考试号> <序号>
python3 build/flatten.py writing 写作 1B 002       # → 整理卷/002_PTE模考1B_写作_整理卷.tex

# 含图片: 末尾多传图片目录名(图片放 build/paper/<目录>/, 正文用 \includegraphics{<目录>/x.png})
python3 build/flatten.py speaking 口语 1B 001 img  # → 整理卷/001_PTE模考1B_口语_整理卷/(含 tex+img/)
```

**② 编译验证 + 比对**(应和工作区 preview 一模一样):
```bash
cd "整理卷/001_PTE模考1B_口语_整理卷"      # 含图片版本进子文件夹; 纯文字版本就在 整理卷/
xelatex 001_PTE模考1B_口语_整理卷.tex && xelatex 001_PTE模考1B_口语_整理卷.tex
rm -f *.aux *.log *.out                     # 只留 .pdf + .tex(+img/)
```

**③ 落盘规则**(务必遵守,见 §A):
- 只有一个文件夹 `整理卷/`;文件名 `<序号>_PTE模考<编号>_<部分>_整理卷`。
- **每个 part 各自从 `000` 起 +1**(不与别的 part 混排)。
- 纯文字→松散两文件;含图片→子文件夹(tex+img/+pdf)。
- **绝不修改 / 覆盖任何旧版本**,只新增。

## A. 版本规则(复述,必须照做)

1. 一个版本 = 一对成品 `<序号>_PTE模考<编号>_<部分>_整理卷` 的 `.pdf` + `.tex`。
2. **只有一个成品文件夹 `整理卷/`**,所有 part、所有版本都放进去,靠文件名里的部分名区分。
3. **每个 part 各自从 `000` 起编号**(写作 000/001…、口语 000/001… 互不干扰),每更新一版 **+1**(三位数)。
4. **旧版本只读,永不修改 / 覆盖**;只新增下一个序号。
5. **纯文字版本**:单文件自包含 `.tex` 直接放 `整理卷/`(写作即如此)。**含图片版本**(如口语 DI 配图):做成**子文件夹** `整理卷/<序号>_…_整理卷/`(内含 `.tex` + `img/` + `.pdf`),整体自包含。两者都用 `build/flatten.py`(含图片时多传图片目录名)。
